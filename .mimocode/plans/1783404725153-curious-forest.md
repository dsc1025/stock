# 股票缓存数据迁移至MySQL计划

## 一、需求概述

将项目中所有基于文件系统的持久化数据迁移到MySQL数据库（localhost:3306），包括：
- 历史K线数据（5492个CSV文件）
- 投资组合数据（portfolio.json）
- 自选股列表（watchlist.txt）
- 选股配置（stock_picker_config.json）
- 策略预设（filters/*.json）

迁移后完全删除CSV文件，所有读写操作改为数据库操作。

## 二、数据库表结构设计

### 2.1 核心表设计（单表方案）

```sql
-- 股票历史K线数据表
CREATE TABLE stock_history (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    code VARCHAR(20) NOT NULL COMMENT '股票代码（如sh.600519）',
    date DATE NOT NULL COMMENT '交易日期',
    open DECIMAL(10,2) COMMENT '开盘价',
    high DECIMAL(10,2) COMMENT '最高价',
    low DECIMAL(10,2) COMMENT '最低价',
    close DECIMAL(10,2) COMMENT '收盘价',
    volume BIGINT COMMENT '成交量（股）',
    amount DECIMAL(18,2) COMMENT '成交额（元）',
    turn DECIMAL(8,4) COMMENT '换手率（%）',
    pctChg DECIMAL(8,4) COMMENT '涨跌幅（%）',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    UNIQUE KEY uk_code_date (code, date),
    INDEX idx_code (code),
    INDEX idx_date (date)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='股票历史K线数据';

-- 投资组合表
CREATE TABLE portfolio (
    id INT AUTO_INCREMENT PRIMARY KEY,
    type ENUM('config', 'position', 'order') NOT NULL COMMENT '数据类型',
    code VARCHAR(20) COMMENT '股票代码（持仓/订单时必填）',
    data JSON NOT NULL COMMENT '详细数据（JSON格式）',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_type (type),
    INDEX idx_code (code)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='投资组合数据';

-- 自选股列表表
CREATE TABLE watchlist (
    id INT AUTO_INCREMENT PRIMARY KEY,
    code VARCHAR(20) NOT NULL COMMENT '股票代码',
    name VARCHAR(50) COMMENT '股票名称',
    added_date DATE COMMENT '添加日期',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE KEY uk_code (code)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='自选股列表';

-- 配置表（存储各种JSON配置）
CREATE TABLE configs (
    id INT AUTO_INCREMENT PRIMARY KEY,
    config_key VARCHAR(50) NOT NULL COMMENT '配置键名',
    config_value JSON NOT NULL COMMENT '配置值（JSON格式）',
    description VARCHAR(200) COMMENT '配置说明',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    UNIQUE KEY uk_key (config_key)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='系统配置表';
```

### 2.2 数据映射关系

| 原始文件 | 目标表 | 转换说明 |
|---------|--------|---------|

### 步骤2：创建数据库配置模块 `db_config.py`
```python
import pymysql
from contextlib import contextmanager

DB_CONFIG = {
    'host': 'localhost',
    'port': 3306,
    'user': 'root',
    'password': '221127',  # 实施时需要用户提供
    'database': 'stock_db',
    'charset': 'utf8mb4',
    'cursorclass': pymysql.cursors.DictCursor
}

@contextmanager
def get_connection():
    conn = pymysql.connect(**DB_CONFIG)
    try:
        yield conn
        conn.commit()
    except Exception:
        conn.rollback()
        raise
    finally:
        conn.close()
```

## 五、关键代码示例

### 5.1 数据库版 `get_stock_history()`
```python
def get_stock_history(code: str, days: int = 60) -> pd.DataFrame:
    """获取历史K线数据。优先级：内存缓存 → 数据库 → API"""
    # 1. 内存缓存
    if code in _hist_cache:
        return _hist_cache[code]
    
    # 2. 数据库
    db_df = db_manager.load_stock_history(code)
    if db_df is not None and not db_df.empty:
        _hist_cache[code] = db_df
        return db_df
    
    # 3. API获取并保存到数据库
    df = _fetch_history_from_api(code, days)
    if not df.empty:
        db_manager.save_stock_history(code, df)
        _hist_cache[code] = df
    return df
```

## 六、注意事项

### 6.1 性能优化
- 批量插入使用 `executemany()` 或 `LOAD DATA INFILE`
- 添加适当的索引（已在表结构中设计）
- 保留内存缓存减少数据库查询

### 6.2 数据安全
- 迁移前备份原始CSV文件
- 迁移脚本支持断点续传（记录已处理文件）
- 提供回滚机制

### 6.3 错误处理
- 数据库连接失败时的降级处理
- 数据格式异常的捕获和日志记录
- 事务保证数据一致性
